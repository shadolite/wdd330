Notes:

  • information stored or shared with the user is called output
  • "If you design for the smallest, least powerful device first, then you will focus on the content, and your core functionality. That leads to simple, beautiful apps. If you do ith the other way around, it'll be like trying to put a marshmallow into a piggy bank, this is neither simple, nor beautiful"
    Joel Marsh - The Hipper Element
  • "Whatever you are doing, do mobile first"
    Google
  • 1. Be there
    2. Be useful
    3. Be quick
  • Speed Matters
    - Reduce image usage
    - SVG/Webfonts are lower bandwidth
    - Optimize and Minify CSS and JS
    - GZIP files to remove unnecessary data
  • Placement Matters
    - Reachability
    - Perception (menu label button instead of hamburger button)
  • Forms
    - Use float labels
    - Only ask for needed information
    - Paginate larger forms and include progress indicator, enable autofill
    - Show passwords by default, allow hiding
    - Auto-advance to next fields
    - Bring up apropriate keyboard for desired input
    - Link phone numbers
  • The way people respond to ads matters most
    - Don't make ads full size to start with
    - Ad windows can display full-sized ads
